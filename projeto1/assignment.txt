Feature Analysis and Engineering 
Objective: 
Explore, clean, transform, and engineer features from the Gold_data.csv dataset. In this 
assignment, students must use Close as the target variable and treat the remaining 
variables as input features. The focus is on understanding and documenting each step  of 
preprocessing, transformation, selection, and engineering, rather than building a full 
predictive model. 
Dataset Description: 
The dataset is provided as a CSV file named Gold_data.csv in this link: Gold_data.csv 
For this assignment the target variable is: “Close” which corresponds to the Gold closing 
price of the day. 
 
Assignment Steps and Reporting Guidelines 
 
1. Data Import and Exploration 
Task: 
Load the Gold_data.csv dataset. 
Report Deliverables: 
• Describe the dataset, including the number of observations, number of variables, 
number of missing values or NaN(s).  
• Clearly state that Close is the target variable and the remaining columns are input 
features and describe the distribution of that variable. 
• Show outputs in a summary table from commands such as head, info, describe, 
size, or equivalent.  
• Display the first few rows of the processed dataset.  
• Discuss initial observations regarding data types, value ranges, and any 
irregularities. 
 
2. Data Cleaning and Preprocessing 
Task: 
Check the input features and target variable for missing, zero, duplicated, or anomalous 
values and handle them appropriately. Apply any necessary preprocessing such as 
imputation, clipping, normalization, format correction or scaling.

3. Exploratory Data Analysis (EDA) 
Task: 
Explore the distributions and relationships between the input features and the target 
variable Close. 
Report Deliverables:

o histograms  
o boxplots

4. Feature Transformation Techniques 
Task: 
Apply suitable feature transformations to the input variables and, if justified, to the target 
variable. 
Report Deliverables: 
• Explain why feature transformation is useful in this dataset.  
• Apply transformations such as z -score normalization, min -max scaling, or log 
transformation where appropriate.  
• Illustrate the effects of transformation using before-and-after visualizations.  
• Discuss whether these transformations improve feature comparability or reduce 
skewness.

5. Feature Selection Methods 
Task: 
Evaluate redundancy among the input features and identify a subset of features that are 
most useful for explaining the target variable Close. 
Report Deliverables: 
• Provide a correlation matrix or heatmap for the input features.  
• Analyze the relationship between each input feature and the target variable.  
• Explain the criteria used to remove redundant or weakly useful features.  
• Present the final selected subset of input features and justify your choices.  
• Discuss why feature selection is important before predictive modeling. 
 
6. Feature Engineering 
Task: 
Create one or more new input features from the original input variables to improve the 
representation of the data. 
Report Deliverables: 
• Engineer at least two new features from the input variables.  
• Explain why the engineered features were created and how they may help explain 
the target variable Close.  
• Include at least one visualization showing the behavior of the engineered features.  
• Discuss whether the new features appear to provide additional useful information. 
 
7. Comparative Analysis and Reporting 
Task: 
Compare the original input features, transformed features, selected features, and 
engineered features with respect to their usefulness for predicting Close. 
Report Deliverables: 
• Provide visual or statistical comparisons between the different feature sets.  
• Summarize the main findings from each step of the assignment.  
• Discuss which preprocessing and engineering steps were most useful.  
• Explain how the final feature set may support future predictive modeling of Close.
